6  From Question to Final Figure

6.1 Learning Goals

By the end of this chapter, the main goals are:

  • move from a panel dataset to a focused cross-section
  • compare a detailed scatterplot with a compressed grouped summary
  • keep labels and axis units aligned with the substantive variables

6.2 Why This Case Study

Cigarette taxes are the single most widely used policy tool for reducing smoking, and they vary enormously across U.S. states — from a few cents per pack in tobacco-producing states to well over a dollar elsewhere. That variation is the kind of policy laboratory social scientists work with all the time: a large, continuous predictor that states set themselves, a clearly measurable outcome (packs per capita), and a real substantive question behind it.

The working question for the rest of the chapter is:

How do state cigarette taxes relate to cigarette consumption, and what is the clearest way to show that relationship?

The tradeoffs that come up answering it — detailed scatterplot versus compressed summary, one year versus two, one state highlighted versus all, a table of numbers versus a figure — recur in almost every piece of descriptive state-level analysis. The case study is small, but the moves it teaches generalize well beyond tobacco policy.

6.3 Load Data

library(tidyverse)  # Core data import, transformation, and plotting tools
library(ggrepel)    # Separate overlapping text labels on plots

options(scipen = 999)  # Print numbers in full, not scientific notation

cigarettes_raw <- read_csv("Data/cigarettes/cigarettesSW.csv", show_col_types = FALSE) |>
  mutate(across(where(is.numeric), round))

This dataset records cigarette consumption and related state-level variables in 1985 and 1995. packs is annual cigarette packs per capita. For taxes and prices, there are two versions of each column: tax and price are nominal cents per pack (the amount printed on the receipt at the time), while real_tax and real_price are the same amounts adjusted for inflation to constant dollars. For any comparison across years — 1985 vs 1995 — the real versions are what matter, because 20 cents in 1985 was worth substantively more than 20 cents in 1995. The rest of the chapter uses real_tax for that reason.

The data form a small panel, with each state observed in two years.

Before narrowing to a single year, it is worth looking at how the tax variable is distributed across states in each year. A density plot is well suited to this: it shows the shape of the distribution without reducing it to a single number, and overlaying both years in one panel makes the shift between them easy to see.

ggplot(cigarettes_raw, aes(x = real_tax, fill = factor(year), color = factor(year))) +
  geom_density(alpha = 0.35, linewidth = 0.8) +
  labs(
    title = "State Cigarette Tax Distributions In 1985 And 1995",
    subtitle = "Densities are overlaid in one panel for direct comparison",
    x = "Cigarette tax (real cents per pack)",
    y = "Density",
    fill = "Year",
    color = "Year"
  ) +
  theme_minimal()

A few things are worth noticing here. year is numeric in the data, so it is wrapped in factor() to treat it as a discrete grouping variable — otherwise ggplot2 would try to give it a continuous color gradient instead of one fill per year. Both fill and color are mapped to the same variable so that each density gets a colored outline as well as a translucent fill, and alpha = 0.35 in geom_density() keeps both years visible where they overlap. The overall pattern is that the 1995 distribution sits noticeably to the right of the 1985 distribution, which means taxes were substantively higher across states a decade later.

The density plot is not the only way to compare these distributions. Two other common choices show the same information with different tradeoffs.

A boxplot compresses each year’s distribution into a small five-number summary: median, quartiles, and whiskers. The result is compact and easy to read when several groups need to be compared side by side.

ggplot(cigarettes_raw, aes(x = factor(year), y = real_tax, fill = factor(year))) +
  geom_boxplot(alpha = 0.5, width = 0.5) +
  labs(
    title = "State Cigarette Taxes By Year",
    subtitle = "Boxplots compress each distribution into a five-number summary",
    x = "Year",
    y = "Cigarette tax (real cents per pack)",
    fill = "Year"
  ) +
  theme_minimal() +
  theme(legend.position = "none")

The legend is suppressed because the x-axis already labels the two groups. Boxplots are good at center-and-spread comparisons but hide the actual shape of the distribution — for example, bimodality or skew would not show up at all.

A violin plot keeps the side-by-side compactness of a boxplot while restoring the shape information from the density plot. Each “violin” is a symmetric density curve, so wider regions indicate more states concentrated at that tax level.

ggplot(cigarettes_raw, aes(x = factor(year), y = real_tax, fill = factor(year))) +
  geom_violin(alpha = 0.5) +
  labs(
    title = "State Cigarette Taxes By Year",
    subtitle = "Violin plots combine the compactness of boxplots with the shape of densities",
    x = "Year",
    y = "Cigarette tax (real cents per pack)",
    fill = "Year"
  ) +
  theme_minimal() +
  theme(legend.position = "none")

The three plots tell the same basic story with different emphasis. Density overlays make the shift between years most obvious; boxplots make the central tendencies easiest to compare numerically; violin plots sit between the two. The choice usually depends on whether the reader cares more about shape, summary statistics, or both.

6.7 Class Exercise: Try A Different Predictor

Time: about ten minutes.

The scatterplot above uses real_tax as the predictor. But the tax is only one part of what smokers actually pay — real_price is the full retail price per pack (inflation-adjusted), and income is average state per-capita income. Either one is a plausible predictor of cigarette consumption, and each tells a different substantive story.

For the 1995 cross-section:

  1. pick either real_price or income as the x-axis variable
  2. keep packs on the y-axis
  3. add a smoother with geom_smooth(method = "lm", formula = y ~ x, se = FALSE)
  4. label the axes in the correct units (real price is in cents per pack, income is per-capita dollars)
  5. add a title that names the predictor, and use theme_minimal()

Then answer briefly:

  • Is the association stronger, weaker, or in a different direction than the tax-vs-packs plot?
  • Which of the three predictors (real_tax, real_price, income) do you think is the most informative, and why?

ggplot(cig_1995, aes(x = ..., y = ...)) +
  geom_point() +
  geom_smooth(...) +
  labs(...) +

6.8 Step 4: Add A Smoother

ggplot(cig_1995, aes(x = real_tax, y = packs)) +
  geom_point() +
  geom_smooth(method = "lm", formula = y ~ x, se = FALSE)

The smoother makes the negative association easier to see. It summarizes the pattern in the cross-section, but it does not by itself establish a causal effect. Higher taxes may reduce smoking, but taxes may also reflect political or cultural differences across states that are related to smoking in other ways.

6.9 Step 5: Add A Title, Axis Labels, And A Theme

The plot from Step 4 shows the pattern but is unlabeled. labs() adds a title, an optional subtitle, and axis labels. theme_minimal() replaces the default gray panel with a lighter look that puts the focus on the data.

ggplot(cig_1995, aes(x = real_tax, y = packs)) +
  geom_point() +
  geom_smooth(method = "lm", formula = y ~ x, se = FALSE) +
  labs(
    title = "Cigarette Taxes And Consumption In 1995",
    subtitle = "The smoother summarizes the overall association",
    x = "Cigarette tax (real cents per pack)",
    y = "Annual cigarette packs per capita"
  ) +
  theme_minimal()

This version is publication-adjacent: the pattern is clear, the axes are legible, and a reader who only sees the figure can still tell what it is about. The default axis ticks are perfectly readable, so there is no need to override them here.

6.10 Step 6: Label The States Directly

The scatterplot shows the overall relationship well, but each point is still anonymous. geom_text_repel() from the ggrepel package adds text labels that are automatically nudged apart so they do not overlap each other or the points.

final_plot <- ggplot(cig_1995, aes(x = real_tax, y = packs)) +
  geom_point() +
  geom_smooth(method = "lm", formula = y ~ x, se = FALSE) +
  geom_text_repel(aes(label = state), seed = 42) +
  scale_x_continuous(breaks = seq(0, 100, by = 10)) +
  labs(
    title = "Cigarette Taxes And Consumption In 1995",
    subtitle = "Each label is a state; the smoother summarizes the overall association",
    x = "Cigarette tax (real cents per pack)",
    y = "Annual cigarette packs per capita"
  ) +
  theme_minimal()

final_plot

geom_text_repel() only needs one mapping — aes(label = state) — to do its job. Everything else has a sensible default: it picks label sizes, nudges overlapping labels apart, and draws short line segments to the points they label. The only other argument here is seed = 42, which makes label placement reproducible — without it, ggrepel would pick slightly different positions on each render.

This is also the first place a custom axis scale earns its keep. With state names crowding the plot area, denser x-axis tick marks help a reader look up a specific state’s tax value: scale_x_continuous(breaks = seq(0, 100, by = 10)) replaces the defaults with a tick every 10 cents. scale_x_continuous() and its partner scale_y_continuous() control tick positions (breaks), axis range (limits), and label formatting (labels) on any continuous axis.

The whole plot is also saved to an R object named final_plot, which will be useful in the export step later.

The main pattern is negative, but not mechanical. Higher-tax states generally cluster at lower levels of cigarette consumption, yet there is still meaningful variation among states with similar tax levels. That is another reason the full scatterplot is more informative than a simple grouped summary.

6.11 Step 7: Does The Same Pattern Hold In 1985?

One natural check is whether the tax-consumption association looks similar a decade earlier. Faceting by year puts the two cross-sections side by side on the same axes, so the overall slope is easy to compare.

comparison_data <- cigarettes_raw |>
  filter(year %in% c(1985, 1995))

ggplot(comparison_data, aes(x = real_tax, y = packs)) +
  geom_point(color = "gray60", alpha = 0.7) +
  geom_smooth(method = "lm", formula = y ~ x, se = FALSE, color = "gray35") +
  facet_wrap(~ year) +
  labs(
    title = "Cigarette Taxes And Consumption In 1985 And 1995",
    x = "Cigarette tax (real cents per pack)",
    y = "Annual packs per capita"
  ) +
  theme_minimal()

The negative slope is visible in both years. Because the x-axis is in real (inflation-adjusted) cents, the change between panels reflects actual policy shifts rather than general price-level drift — taxes rose substantively in real terms, and consumption dropped. The two-year cigarettesSW file is enough for a simple cross-section and a two-panel comparison, but it is not ideal for a fuller time-series exercise. For that, the longer cigarette panel included with the course materials is more useful.

6.12 Step 8: A Longer Time-Series View

The cigarette.csv file in the course materials is the same kind of data but covers many more years. With it the question can shift from “what is the cross-section in 1995?” to “how did consumption in specific states change over time?”

long_cigarettes <- read_csv("Data/cigarettes/cigarette.csv", show_col_types = FALSE) |>
  mutate(across(where(is.numeric), round)) |>
  mutate(year_date = as.Date(paste0(year, "-01-01")))

ggplot(
  long_cigarettes |> filter(state %in% c("NC", "CA", "TX", "NY")),
  aes(x = year_date, y = packpc, color = state)
) +
  geom_line() +
  facet_wrap(~ forcats::fct_reorder(state, packpc, .fun = mean)) +
  scale_x_date(date_breaks = "2 years", date_labels = "%Y") +
  labs(
    title = "Cigarette Consumption In Four States",
    x = NULL,
    y = "Annual packs per capita"
  ) +
  theme_minimal() +
  theme(legend.position = "none")

The legend is suppressed because each facet header already names the state. Faceting works well when a handful of specific states deserve direct comparison and one shared panel would be too crowded. The panels are ordered by average consumption, which makes the ranking immediately visible — North Carolina at the top, California at the bottom. All four states trend downward, but the levels differ by a factor of roughly two.

scale_x_date(date_breaks = "2 years", date_labels = "%Y") is the first date scale in this chapter. It tells ggplot2 to treat the x-axis as dates, place tick marks every two years, and print the labels as four-digit years.

6.13 Step 9: Build A More Compressed Summary

cig_tax_groups <- cig_1995 |>
  mutate(
    tax_group = factor(
      ntile(real_tax, 3),
      levels = c(1, 2, 3),
      labels = c("Lower tax", "Middle tax", "Higher tax")
    )
  ) |>
  group_by(tax_group) |>
  summarize(
    mean_packs = round(mean(packs, na.rm = TRUE)),
    mean_real_tax = round(mean(real_tax, na.rm = TRUE)),
    .groups = "drop"
  )

6.15 Step 11: Choose The Final Figure

Sometimes the cleanest way to argue for one view over another is to show them together. The patchwork package makes this simple: load it, and then use the + operator to glue two ggplot objects into a single figure.

library(patchwork)

combined_plot <- (final_plot + summary_plot) +
  plot_annotation(
    title = "Two Views Of The Same 1995 Cigarette-Tax Data",
    subtitle = "The labeled scatterplot preserves state-level detail; the bar chart compresses it into three groups"
  )
combined_plot

Because both plots were saved as R objects in earlier steps (final_plot in Step 6, summary_plot in Step 10), combining them is a one-liner. plot_annotation() adds a title and subtitle that sit above the whole figure rather than either panel. patchwork supports richer layouts too — / stacks plots vertically, | puts them side by side, and parentheses group them — but for two plots the plain + is enough. Seeing the detailed scatterplot next to the compressed bar chart makes the tradeoff concrete: the bar chart is easier to summarize verbally, but it throws away almost everything the scatterplot shows. For the original question, the labeled scatterplot is the better final figure — it preserves the continuous relationship between tax and consumption and keeps the states identifiable.

6.16 Step 12: Finalize And Export

ggsave("final_cigarettes_plot.png", plot = final_plot, width = 8, height = 5, dpi = 300)
ggsave("final_cigarettes_plot.pdf", plot = final_plot, width = 8, height = 5)

Saving the figure as an object before export is a small but durable habit. It keeps the workflow readable and makes it easy to reuse the same figure for multiple output formats or sizes.

6.17 Reflection

The core workflow was:

  1. inspect the panel structure
  2. narrow the question to one year
  3. build a first scatterplot
  4. add a smoother and direct labels
  5. compare that plot with a compressed summary
  6. choose the figure that best matches the question

That sequence is common in real work. A useful figure rarely appears fully formed on the first try. It is usually the result of several narrowing and revision decisions.

6.19 Extra Material

6.19.1 Extra 1: Highlight One State Against All Others

The long_cigarettes object built in Step 7 is reused here. The key move is to draw every state in light gray, then draw one state a second time in black on top of the gray lines.

ggplot(long_cigarettes, aes(x = year_date, y = packpc, group = state)) +
  geom_line(color = "gray80", alpha = 0.5) +
  geom_line(
    data = long_cigarettes |> filter(state == "NC"),
    color = "black",
    linewidth = 1
  ) +
  scale_x_date(date_breaks = "2 years", date_labels = "%Y") +
  labs(
    title = "Cigarette Consumption In North Carolina Highlighted",
    x = NULL,
    y = "Annual packs per capita"
  ) +
  theme_minimal()

This is one of the cleanest highlighting patterns for time series: keep all states for context, but push them into the background so one state becomes easy to follow.

6.19.2 Extra 2: A Very Simple US State Map

Mapping deserves its own course, but a first state-level map can be made with almost no extra code using the usmap package. plot_usmap() handles the projection, state boundaries, and aesthetics in a single call — the only required inputs are a data frame with a state column (two-letter abbreviations or full names) and the name of the value column to shade by.

library(usmap)

plot_usmap(data = cig_1995, values = "real_tax", color = "white",
           exclude = c("AK", "HI")) +
  scale_fill_continuous(
    low = "gray90",
    high = "firebrick",
    name = "Cigarette tax\n(real cents per pack)",
    label = scales::label_number()
  ) +
  labs(
    title = "Cigarette Taxes By State In 1995",
    subtitle = "Darker states have higher taxes"
  ) +
  theme(legend.position = "right")

A few things are worth noting. plot_usmap() returns a regular ggplot object, so it can be extended with scale_fill_continuous(), labs(), and theme() exactly like any other plot built in this course. Alaska and Hawaii are excluded explicitly with exclude = c("AK", "HI"), keeping the map to the contiguous 48 states. The chunk is marked eval: false so the book renders even when usmap is not yet installed; install.packages("usmap") enables it.

For most research figures, the lesson from the rest of this chapter still applies: a labeled scatterplot usually answers a quantitative question better than a map. But a map is often the right choice when geographic pattern itself is the point — for example, whether high-tax states cluster regionally.

6.19.3 Extra 3: A Hex-Tile Version Of The Same Map

A geographic choropleth has a well-known distortion problem: large low-population states (Alaska, Montana, Wyoming) dominate the visual field while small high-population states (Rhode Island, Connecticut) almost disappear. A hex-tile map assigns every state an equal-area hexagon, so the color patterns reflect policy variation rather than state size.

The statebins package makes this straightforward. Its geom_statebins() works inside a standard ggplot() call with a state aesthetic, so the color scale, labels, and theme all work exactly as elsewhere in this course.

library(statebins)

ggplot(cig_1995, aes(state = state, fill = real_tax)) +
  geom_statebins() +
  scale_fill_continuous(
    low = "gray90",
    high = "firebrick",
    name = "Cigarette tax\n(real cents per pack)"
  ) +
  labs(
    title = "Cigarette Taxes By State In 1995",
    subtitle = "Each tile is one state — equal area removes the large-state distortion"
  ) +
  theme_statebins()

Comparing this to the usmap choropleth for the same data is a good exercise: the geographic shape is gone, but the regional clustering (if any) is usually easier to read once every state occupies the same space. Install with install.packages("statebins").
